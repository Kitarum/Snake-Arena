🧩 ВИЗУАЛ И АССЕТЫ (ключевые принципы)

Цель: иметь возможность начинать с примитивов и постепенно переходить к полноценному визуалу без переписывания логики.

Базовый принцип:
- ЛОГИКА отдельно (позиция, скорость, коллизии, правила)
- ВИЗУАЛ отдельно (спрайт/модель/материал/анимации)

Этапы развития:
1) Примитивы (квадраты/капсулы) → быстрый прототип
2) 2D спрайты + анимации → визуальное улучшение без изменения логики
3) 3D low-poly (при переходе на движок) → модель + материал + свет + анимация

Важно:
Замена квадрата на модель должна быть заменой визуальной оболочки, а не переписыванием механики.

Ассеты:
- Рисовать самому необязательно
- Можно использовать готовые ассеты (с лицензиями)
- Можно собирать модели из частей (kitbash)
- Стиль должен соответствовать возможностям железа

📈 СТРАТЕГИЯ РЫНКА И ДЕНЕГ

Цель:
- доход + творческая свобода + устойчивый рост
- без выгорания и многолетних соло-долгостроев

Основной фокус:

Не основной фокус:

- Telegram как основной источник дохода (подходит для экспериментов и теста механик)

Оптимальный масштаб:
- не гигантский проект на 2–4 года
- не хаотичные мелкие игры
- а mid-scale проект с одной сильной core-механикой и системной глубиной

Подходящие направления:
- roguelike / survivors-like
- automation-lite
- sandbox-lite
- management/simulation

Учитывать:
- особенности выплат из РФ
- необходимость продумывать монетизацию заранее
